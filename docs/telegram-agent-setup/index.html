Guía: Telegram Agent UI Phase 3

Esta guía te mostrará cómo crear un proyecto completo que integra un bot de Telegram con una interfaz web inteligente y multicanal.

🎯 Objetivo del Proyecto​

Crear un sistema progresivo de un bot agente que demuestre:

  1. Bot con comandos predefinidos (Phase 1)
  2. Bot agente inteligente que entiende lenguaje natural (Phase 2)
  3. Sistema multicanal integrado (Phase 3): Telegram + Web UI + Chat asistente

🏗️ Arquitectura del Proyecto​

El proyecto utiliza una arquitectura unificada donde tanto la UI web como el bot de Telegram comparten la misma lógica de negocio:

┌─────────────────────────────────────────┐
│   Telegram Bot    │    Web UI    │ Chat │
├───────────────────┴──────────────┴──────┤
│         AgentOrchestrator (cerebro)     │
├─────────────────────────────────────────┤
│    InMemoryProjectWorkspaceService      │
│         (dominio + datos)               │
└─────────────────────────────────────────┘

Ventajas:

  • ✅ Sin duplicación de lógica entre canales
  • ✅ Los cambios se aplican automáticamente en todos los canales
  • ✅ Datos y estado sincronizados en tiempo real
  • ✅ Experiencias consistentes en todos los canales

📦 Archivos Principales​

proyecto/
├── bot/
│   └── TelegramAgentBot.java          # Canal Telegram
├── agent/
│   └── AgentOrchestrator.java         # Cerebro del asistente
├── service/
│   └── InMemoryProjectWorkspaceService.java
├── controller/
│   ├── TaskController.java            # API REST de tareas
│   └── AssistantController.java       # API REST del chat
├── src/main/resources/static/
│   ├── index.html                     # UI web
│   ├── app.js                         # Frontend
│   └── styles.css                     # Estilos
└── src/main/resources/
    └── application.properties         # Configuración

🚀 Configuración Inicial​

Paso 1: Obtén el Token de Telegram​

  1. Abre BotFather en Telegram
  2. Crea un nuevo bot con /newbot
  3. Recibirás un token: 123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
  4. Anota el nombre y token

Paso 2: Configurar el Proyecto​

Entra al directorio del proyecto:

cd starter/telegram-agent-ui-phase3

Copia el archivo de propiedades:

cp src/main/resources/application.properties.example src/main/resources/application.properties

Paso 3: Editar las Propiedades​

Abre src/main/resources/application.properties y configura:

# Configuración obligatoria del bot
telegram.bot.name=MiOraclePlannerBot
telegram.bot.token=TU_TOKEN_AQUI

Paso 4: Configuración Opcional (IA/LLM)​

Para habilitar respuestas más inteligentes con OpenAI:

# Habilitar procesamiento con LLM
agent.ai.enabled=true
agent.ai.base-url=https://api.openai.com/v1
agent.ai.api-key=TU_API_KEY_OPENAI
agent.ai.model=gpt-4o-mini

💡 Si no configuras esto, el bot funcionará con reglas predefinidas pero sin procesamiento de lenguaje natural con IA.

⚙️ Ejecutar Localmente​

Una vez configurado, inicia el servidor:

mvn spring-boot:run

El servidor iniciará en http://localhost:8080

🧪 Probar el Sistema​

Interfaz Web​

  1. Abre en tu navegador: http://localhost:8080
  2. Verás:
    • Panel de tareas con vista clara del estado
    • Formulario para crear nuevas tareas
    • Chat asistente integrado

Bot de Telegram​

  1. Abre Telegram
  2. Busca tu bot: @MiOraclePlannerBot (reemplaza con tu nombre)
  3. Puedes escribir de forma natural:
    • "crear una tarea de revisar código"
    • "mostrar mis tareas"
    • "marcar como completada la primera tarea"

Pasos Detallados​

1️⃣ Crear Tareas desde la UI Web

  • Ve a http://localhost:8080
  • Usa el formulario para crear varias tareas
  • Completa los campos: título, descripción, prioridad

2️⃣ Consultar desde el Chat Web

  • En la misma página, usa el chat asistente
  • Pregunta: "¿Cuántas tareas tengo?" o "mostrar tareas de alta prioridad"
  • El asistente responde con contexto de la UI

3️⃣ Consultar desde Telegram

  • Abre el bot en Telegram
  • Escribe las mismas preguntas
  • Verás que el bot conoce las tareas creadas en la web

4️⃣ Explicar la Arquitectura

  • Muestra que ambos canales usan el mismo AgentOrchestrator
  • Explica que no hay duplicación de código
  • Los cambios en una interfaz se reflejan automáticamente en la otra

5️⃣ Realizar Cambios Sincronizados

  • Crea una tarea en Telegram
  • Vuelve a la UI web y verás la tarea creada
  • Completa una tarea desde la web
  • Vuelve a Telegram y verifica que aparece completada

✨ Características Principales​

Panel Visual de Tareas​

  • ✅ Vista clara del estado, prioridad y asignación
  • ✅ Formulario intuitivo sin necesidad de comandos
  • ✅ Actualizaciones en tiempo real
  • ✅ Mejor visualización que Telegram

Bot Inteligente de Telegram​

  • ✅ Comprende oraciones completas en lenguaje natural
  • ✅ No requiere memorizar comandos especiales
  • ✅ Accesible desde cualquier dispositivo
  • ✅ Sincronización en tiempo real con la web

Chat Asistente​

  • ✅ Responde preguntas sobre tareas
  • ✅ Entiende contexto del sistema
  • ✅ Proporciona ayuda sin errores de sintaxis
  • ✅ Integrado en la misma interfaz

📝 Ejemplo de Uso​

Crear Tarea desde Web UI​

Título: Revisar PR del backend
Descripción: Revisar cambios de autenticación
Prioridad: Alta
Sprint: Sprint 1

Ver desde Telegram​

Usuario: ¿qué tareas de alta prioridad tengo?

Bot: Tienes 1 tarea de alta prioridad:
- "Revisar PR del backend" (asignada a ti)

Editar desde Web UI​

Marcar tarea como "Completada"

Verificar desde Telegram​

Usuario: dime el estado de la tarea de revisar PR

Bot: La tarea "Revisar PR del backend" está:
- Estado: Completada
- Prioridad: Alta
- Completada el: 2026-04-30

🐛 Solución de Problemas​

El bot no responde​

  • Verifica que el token es correcto en application.properties
  • Asegúrate que el servidor está ejecutándose (mvn spring-boot:run)
  • Reinicia el bot con /start

Los cambios no se sincronizan​

  • Comprueba que el AgentOrchestrator está inyectado correctamente
  • Verifica los logs del servidor

La UI web no carga​

  • Ve a http://localhost:8080
  • Abre la consola del navegador (F12) para ver errores
  • Comprueba que el puerto 8080 no está ocupado

🎉 ¿Qué Aprendiste?​

Con este proyecto puedes demostrar:

  1. Multicanal: Un mismo servicio en múltiples interfaces
  2. Sin duplicación: La misma lógica para diferentes canales
  3. Sincronización: Datos consistentes en tiempo real
  4. Escalabilidad: Fácil agregar nuevos canales sin duplicar código
  5. Arquitectura limpia: Separación clara de responsabilidades
